# 2026년 김해시 정책제안 공모전 제출본

## 제안명

**김해행정마찰지도**

검색어와 전화돌림 신호로 시민이 막히는 지점을 찾아 매달 실제로 고치는 12주 행정개선 실험

## 1. 공모서식용 요약

김해시 누리집의 인기 검색어·검색량과 민원콜센터의 총응답·전화돌림 집계를 결합해 시민이 정보를 찾거나 담당부서에 연결되는 과정에서 반복적으로 막히는 지점을 찾는다. 월별 상위 후보를 담당부서가 직접 검증하고, 신규 시스템 구축 없이 검색동의어·메뉴·안내문·연결표·상담 매뉴얼을 수정한다. 수정 전후의 검색 도달성, 전화돌림, 처리기간을 같은 기준으로 비교해 효과가 확인된 개선은 표준 운영절차로 남긴다. 12주 동안 3~5개 항목만 실증하며 원시 검색로그, 상담 메모, 녹취 등 개인정보 자료는 사용하지 않는다.

## 2. 제안 배경과 문제점

김해시는 누리집과 민원콜센터를 각각 운영하고 있다. 두 채널에는 시민이 어느 행정정보에서 자주 멈추는지 보여주는 신호가 남지만, 현재 공개자료만으로는 이 신호를 함께 보고 실제 안내체계를 수정한 뒤 효과까지 다시 측정하는 정례 절차를 확인하기 어렵다.

누리집에서는 인기 검색어와 검색량이 공개된다. 그러나 검색량이 많다는 사실만으로 정보가 부족하거나 찾기 어렵다고 단정할 수 없다. 단순 관심 증가, 계절성 사업, 홍보 효과일 수도 있으므로 담당자의 원인 검증이 필요하다.

민원콜센터의 2024년 자체처리율은 61.6%였다. 성과보고서의 산정방식에 따르면 나머지 38.4%는 전체 전화돌림 비중으로 볼 수 있지만, 이를 오이관율이나 재이관율로 해석해서는 안 된다. 전화돌림 자체가 필요한 전문민원도 있기 때문이다.

핵심 문제는 데이터 부족이 아니라 **이미 존재하는 신호가 부서별 보고에서 끝나고, 작은 행정개선과 전후 측정으로 연결되는 공통 절차가 없다는 점**이다.

## 3. 제안 내용

### 3.1 행정마찰의 정의

행정마찰은 시민이 서비스를 이용하는 과정에서 불필요하게 반복 검색하거나, 안내를 다시 읽거나, 여러 부서로 전화를 돌려야 하는 상태를 뜻한다.

`김해행정마찰지도`의 지도는 지리정보시스템이 아니다. 시민 여정의 단계, 이용 채널, 담당부서별로 마찰 후보와 개선 상태를 한 장에 표시하는 월간 업무용 표이다. 별도 플랫폼 대신 기존 스프레드시트나 업무보고 양식으로 시작한다.

### 3.2 기본 운영 구조

1. **신호 수집**: 월별 인기 검색어·검색량과 콜센터 총응답·전화돌림 집계를 모은다.
2. **후보 정리**: 같은 업무를 가리키는 표현을 묶고 반복성, 두 채널의 동시 발생 여부, 수정 가능성을 표시한다.
3. **담당자 검증**: 데이터만으로 불편을 단정하지 않고 실제 업무 담당자가 원인을 확인한다.
4. **작은 수정**: 검색동의어, 메뉴명, 안내문, 자주 묻는 질문, 담당부서 연결표, 상담 매뉴얼을 고친다.
5. **전후 비교**: 동일한 기준으로 검색 도달성, 전화돌림, 조치기간을 다시 측정한다.
6. **표준화**: 효과가 확인된 변경은 유지하고, 효과가 없으면 원인을 기록해 되돌리거나 다시 수정한다.

### 3.3 후보 우선순위

후보는 아래 네 항목을 합산한 10점 척도로 정렬하되, 최종 선정은 담당자가 결정한다.

| 항목 | 배점 | 판단 기준 |
|---|---:|---|
| 발생 규모 | 0~3 | 검색량 또는 전화돌림 집계가 상대적으로 큰가 |
| 지속성 | 0~2 | 한 번의 행사성이 아니라 반복되는가 |
| 교차 신호 | 0~2 | 검색과 콜센터 양쪽에서 같은 업무가 나타나는가 |
| 수정 가능성 | 0~3 | 2주 안에 메뉴·문구·동의어·연결표로 개선할 수 있는가 |

점수는 시민 불편을 자동 판정하는 알고리즘이 아니라 검토 순서를 정하기 위한 보조수단이다.

### 3.4 가상 적용 예시

특정 증명서의 공식 명칭과 시민이 사용하는 표현이 달라 검색이 반복되고, 콜센터에서도 담당부서 문의가 이어진다고 가정한다. 담당자는 실제 검색결과와 연결 절차를 확인한 뒤 시민 표현을 검색동의어로 등록하고, 메뉴명과 발급 안내를 수정하며, 콜센터 연결표에 담당업무를 명확히 적는다. 이후 같은 표현의 공식 페이지 도달 여부와 관련 전화돌림 변화를 비교한다.

이 예시는 적용방식을 설명하기 위한 가상 사례이며 실제 김해시 데이터에서 확인된 민원 사례가 아니다.

## 4. 12주 실증계획

| 기간 | 수행 내용 | 산출물 |
|---|---|---|
| 1~2주 | 담당부서 지정, 데이터 정의, 개인정보·보안 검토 | 데이터 항목표, 역할표 |
| 3~4주 | 최소 데이터 수집, 기준선 작성 | 최초 마찰지도, 기준선 |
| 5주 | 상위 후보 검증 및 3~5개 선정 | 원인확인 기록, 개선목록 |
| 6~7주 | 메뉴·문구·검색동의어·연결표·매뉴얼 수정 | 변경 전후 기록 |
| 8~11주 | 같은 기준으로 사후 측정, 외부요인 기록 | 사후 지표, 예외 기록 |
| 12주 | 효과 판정, 유지·보완·중단 결정 | 결과보고서, 표준 운영안 |

운영회의는 월 1회 30분을 기본으로 하고, 각 개선 항목에 담당자·완료예정일·변경내용·결과를 남긴다.

## 5. 데이터 사용 범위

### 5.1 반드시 필요한 최소 데이터

| 채널 | 항목 | 활용 |
|---|---|---|
| 누리집 | 월별 인기 검색어 | 시민이 사용하는 표현 파악 |
| 누리집 | 월별 검색량 | 반복 규모와 지속성 확인 |
| 콜센터 | 월별 총응답 건수 | 전화돌림 비율의 분모 |
| 콜센터 | 월별 전화돌림 건수 | 전체 연결 마찰의 추세 확인 |
| 업무관리 | 검증 결과·변경내용·완료일 | 분석이 실제 개선으로 이어졌는지 관리 |

### 5.2 있을 때만 사용하는 선택 데이터

- 검색결과 0건
- 검색결과 클릭
- 연속 재검색
- 표준화된 문의유형
- 최초 이관 부서
- 재이관 여부와 횟수

선택 데이터가 없으면 새 수집기능을 개발하지 않고 최소 데이터만으로 실증한다.

### 5.3 사용하지 않는 데이터

- 이름, 전화번호, 주소 등 개인식별정보
- 상담 메모 원문
- 통화 녹취
- 개인별 검색 이력
- 개인을 역추적할 수 있는 소수 건수

검색어에도 개인정보가 포함될 수 있으므로 원문 전체를 제안 운영팀에 넘기지 않는다. 기존 시스템 담당부서가 월별 집계·표준화·마스킹을 수행하고, 정보보호팀이 정한 소수빈도 기준을 적용한다.

## 6. 성과 측정

### 6.1 실행 지표

| 지표 | 산식·확인 방법 |
|---|---|
| 검증 완료율 | 원인 검증 완료 후보 수 ÷ 검토 후보 수 |
| 개선 완료율 | 실제 수정 완료 항목 수 ÷ 선정 항목 수 |
| 평균 개선기간 | 선정일부터 수정 완료일까지 평균 일수 |
| 근거 기록률 | 변경 전후 지표와 판단 사유를 모두 남긴 항목 비율 |

### 6.2 시민 체감 지표

| 지표 | 적용 조건 | 해석 |
|---|---|---|
| 공식 페이지 도달성 | 대상 검색어를 수동 점검 | 상위 검색결과에 정확한 공식 안내가 나타나는지 확인 |
| 전화돌림 변화 | 같은 범위의 집계가 가능할 때 | 전체 또는 해당 문의유형의 전후 변화를 확인 |
| 재검색·0건 변화 | 선택 데이터가 있을 때 | 검색 마찰의 직접 신호로 사용 |
| 간단 과업 성공률 | 동일한 시민 과업을 전후 점검 | 안내를 보고 목적 페이지·담당부서를 찾을 수 있는지 확인 |

검색량 감소 자체를 성공으로 단정하지 않는다. 제도 홍보로 검색량이 늘어도 정확한 페이지에 도달하면 개선일 수 있다. 전화돌림 감소도 전문상담이 필요한 업무에서는 목표가 아닐 수 있으므로 담당자의 해석을 함께 기록한다.

정량 개선목표는 기준선 확인 후 정한다. 근거 없이 `20% 개선`과 같은 수치를 먼저 약속하지 않는다.

## 7. 추진체계

| 역할 | 담당 후보 | 주요 업무 |
|---|---|---|
| 실증 총괄·분석 | AI정책과 데이터융합팀 | 데이터 정의, 후보 분석, 마찰지도 운영, 결과보고 |
| AI 활용 검토 | AI정책과 AI전략팀 | 익명 집계의 분류·요약을 기존 공통기반에서 지원 가능한지 검토 |
| 누리집 데이터·수정 | 정보통신과 정보기획팀 | 검색 집계, 메뉴·안내문·검색동의어 수정 |
| 개인정보·보안 | 정보통신과 정보보호팀 | 익명화, 접근권한, 소수빈도 기준 검토 |
| 콜센터 데이터·수정 | 소통공보관 시민소통팀 | 전화돌림 집계, 연결표·상담 매뉴얼 수정 |
| 원인 검증 | 선정 업무의 소관부서 | 실제 원인 확인, 업무내용 수정, 효과 해석 |

위 부서는 공개 업무분장에 따른 담당 후보이며, 실제 주관·협조 관계는 실증 착수 전에 확정한다.

## 8. 실행 가능성과 비용

김해시는 2026년 누리집 통합 유지관리 용역을 운영 중이고, 2025년 민원콜센터 상담시스템 재구축을 완료했으며, AI정책과 감독의 데이터 분석 사업 수행 이력이 있다. 따라서 새 플랫폼을 만드는 대신 현재 시스템과 운영조직을 연결하는 방식이 적합하다.

### 최소형 비용 원칙

- 신규 앱·대시보드·상담시스템을 구축하지 않는다.
- 공개 인기 검색어와 기존 월별 콜센터 집계를 사용한다.
- 마찰지도는 기존 스프레드시트나 업무보고 양식으로 작성한다.
- 기존 유지관리 범위의 메뉴·문구·검색동의어·연결표 수정부터 시행한다.
- 추가 외주개발비 `0원`을 목표로 하되 내부 인력 투입은 비용이 없는 것으로 표현하지 않는다.
- 데이터 추출이나 설정 변경이 기존 계약 범위를 벗어나면 비용을 임의 산정하지 않고 해당 기능을 제외하거나 별도 공수를 확인한다.

기존 계약 총액은 전체 시스템의 유지관리·재구축·분석사업 비용이므로 본 파일럿 사업비로 인용하지 않는다.

## 9. 기존 사업과의 차별성

| 일반적인 유사 사업 | 김해행정마찰지도 |
|---|---|
| AI 챗봇 또는 새 상담채널 구축 | 기존 누리집·콜센터 신호 활용 |
| 민원 자동분류·자동배정 | 담당자의 원인 검증과 작은 운영 수정 |
| 단일 채널 통계 보고 | 검색과 전화돌림의 교차 확인 |
| 분석 보고서 제출로 종료 | 수정·전후 측정·유지 여부 결정까지 한 주기 |
| 대규모 시스템 개발 | 12주, 3~5개 항목의 최소 실증 |

본 제안의 신규성은 AI 기술 자체가 아니라 **분석 결과가 실제 행정정보 수정으로 닫히는 운영 루프**에 있다. AI는 개인정보가 제거된 집계의 분류·요약을 돕는 선택수단이며 필수 전제가 아니다.

## 10. 기대효과

### 시민

- 필요한 정보를 찾기 위한 반복 검색과 불필요한 전화 연결을 줄인다.
- 시민이 사용하는 표현을 행정 메뉴와 안내문에 반영한다.
- 새 서비스를 배우지 않아도 기존 누리집과 콜센터에서 개선을 체감할 수 있다.

### 행정

- 부서별로 흩어진 운영 신호를 공통 개선과제로 전환한다.
- 큰 시스템 구축 전에 작은 변경의 효과를 검증해 예산 위험을 낮춘다.
- 어떤 안내를 왜 바꿨고 효과가 있었는지 근거를 축적한다.
- 효과가 확인된 절차를 다른 민원업무와 부서에 반복 적용할 수 있다.

| 검색량이 불편을 의미하지 않음 | 담당자 검증 없이는 개선대상으로 확정하지 않음 |
| 전화돌림을 오이관으로 오해 | 전체 연결 신호로만 사용하고 세부 판정은 데이터가 있을 때만 수행 |
| 세부 로그가 없음 | 공개 검색량과 기존 총응답·전화돌림만으로 최소 실증 |
| 개인정보 노출 | 원문·녹취·상담메모 제외, 월별 익명 집계와 소수빈도 억제 |
| 외부요인으로 전후 지표 변동 | 행사, 제도변경, 홍보, 계절성을 결과표에 함께 기록 |
| 분석만 하고 수정하지 않음 | 개선 담당자·기한·완료 여부를 필수 산출물로 관리 |
| 유지관리 계약 밖의 개발 필요 | 신규 기능은 후속 고도화로 넘기고 파일럿 범위 축소 |
