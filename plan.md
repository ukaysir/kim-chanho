### 고정 작동 원리

`신호 집계 → 개인정보 제거 → AI 목적군 분석 → 담당자 검수 → 월 1건 실제 수정 → 전후 지표 확인`

### 변경하지 않을 원칙

- 다른 아이디어로 전환하지 않는다.
- A1 `민원 인계패스`는 별도 제안으로 합치지 않는다. 필요하면 행정마찰지도가 찾아낸 전화 마찰의 **개선 예시**로만 사용한다.
- `행정마찰지도`를 단순 대시보드나 지리 지도로 설명하지 않는다. 핵심은 **분석 결과로 매달 시민 접점을 실제 수정하는 운영 절차**다.
- 새 시민용 앱이나 대규모 독립 플랫폼을 전제로 하지 않는다.
- 개인별 이용 경로 결합, 통화 녹취 분석, AI의 민원 처분·담당부서 자동 결정, 직원 성과평가는 범위에서 제외한다.
- 세부 로그가 부족하면 아이디어를 바꾸지 않고 시범 범위를 줄인다. 기존 집계 데이터부터 사용하고, 필요한 최소 로그의 추가 여부를 실행 조건으로 적는다.

## 3. 가장 먼저 할 다음 작업

실행을 시작할 때의 첫 작업은 **P0 중복·공고 확인과 P1 데이터 보유 확인**이다. 이 답이 제안서의 실현가능성, 지표, 사업비 표현을 결정하므로 문장 윤문이나 HWPX 입력보다 먼저 수행한다.

### 확인 질문

1. 정책기획과 미래전략팀
   - 공고 이후 정정·연장 공지가 있는가?
   - 검색·전화의 사전 행정마찰을 묶어 정기적으로 실제 서비스를 수정하는 동일·유사 사업이나 채택 제안이 있는가?
   - 2025년 데이터 분석 사업과 2026년 추진사업에 유사 과제가 포함되어 있는가?
2. 누리집·정보통신 담당부서
   - 통합검색에 검색어·검색 횟수 외 결과 수, 결과 클릭, 재검색 항목이 남는가?
   - 개인정보를 제거한 월 단위 집계와 목적군 분석이 가능한가?
3. 콜센터 운영 담당부서
   - 문의 분야, 최초 담당부서 전환, 재전환을 집계할 수 있는가?
   - 생활민원 한 분야를 골라 수정 전후를 같은 기준으로 비교할 수 있는가?
4. 제안자
   - 동일·유사 아이디어를 다른 행정청이나 공모전에 제출한 적이 있는가?
   - 단독 또는 팀 제출인지, 팀이면 기여도와 업무분담은 어떻게 되는가?

확인 결과는 `확인 / 미확인 / 답변 불가`로 구분하고 확인일·확인처·답변 요지를 남긴다. 답변을 받지 못한 사항은 사실처럼 쓰지 않고 실행 전 확인 조건으로 남긴다.

## 5. 초안 확정 시 잠글 내용

### 데이터 범위

- 1단계는 공개자료로 존재가 확인된 검색어 집계와 콜센터 부서 전환 집계를 중심으로 쓴다.
- 검색 결과 0건, 클릭, 재검색, 재전환은 실제 보유 여부가 확인된 항목만 확정 지표로 사용한다.
- 보완 요청·재방문·여러 시스템 간 개인 연결은 초기 시범에서 제외한다.

### AI와 사람의 역할

- AI: 오탈자·동의어·자연어 검색을 같은 행정 목적 후보로 묶고 원인 후보를 제시한다.
- 담당자: 원자료 표본을 검수하고 개선 과제를 확정하며 실제 화면·문구·매뉴얼·연결표를 수정한다.
- AI가 민원 처분이나 담당부서 결정을 대신한다는 표현은 사용하지 않는다.

### 지표와 효과 표현

- 기준선 확인 전 목표 개선율을 만들지 않는다.
- 콜센터 자체 해결률 61.6%에서 역산한 38.4%는 **전체 담당부서 전환 비율**이며, 잘못된 연결 비율이라고 쓰지 않는다.
- 기대효과는 `감소한다`로 단정하지 않고 `수정 전후를 비교해 변화량을 확인한다`로 쓴다.
- 사업비는 로그 항목, 유지관리 계약, AI 공통기반 활용 가능성을 확인한 뒤 산정한다.

### 시각자료 원칙

- 현재 제작한 작동방식 흐름도 1장을 기본 추가자료로 사용한다.
- 이미지에는 `제안 단계의 개념 흐름도이며 실제 운영 화면이나 검증된 성과가 아님`이라는 표시를 유지한다.
- 두 번째 이미지는 새로운 심사 질문에 답할 때만 만든다. 장식 목적의 사진은 추가하지 않는다.
