2026년 김해시 정책제안 공모전 초안

제안명
김해행정마찰지도

부제
누리집에서 막힌 시민의 신호를 찾아 실제 안내를 고치는 12주 실증안

한 문장 제안
김해행정마찰지도는 누리집 안내만으로 해결 가능한 업무가 정보 탐색이나 이해의 어려움 때문에 전화 문의로 넘어가는지 살피는 행정개선 실증안이다. 누리집 인기검색어·조회수와 개인정보를 제거한 콜센터 문의유형별 월 집계를 대조해 개선 후보를 찾는다. 문의유형별 집계 확보를 본 실증의 착수 조건으로 두며, AI는 개인정보 없는 표현을 의미별로 묶는 데만 활용한다. 담당자는 실제 원인을 확인한 뒤 검색·메뉴·안내문·연결체계를 고친다.

1. 문제점 - 답은 있지만 답에 닿지 못한다

누리집에 행정정보를 올렸다고 시민에게 곧바로 전달되는 것은 아니다. 시민의 표현이 행정용어와 어긋나거나 메뉴가 낯설고 안내문에서 다음 행동을 찾기 어려우면 검색은 길어진다. 끝내 답을 찾지 못한 시민은 콜센터에 문의할 수 있다. 온라인에서 마칠 일이 전화 상담과 담당부서 연결로 넘어가는 순간이다.

이 경로가 김해시에서 얼마나 자주 생기는지는 아직 확인되지 않았다. 현재 공개자료만으로는 누리집 검색과 전화 문의의 선후 관계도 파악하기 어렵다. 김해행정마찰지도는 시민 개인의 행동을 연결하지 않는다. 두 채널의 월별 집계에서 같은 업무가 반복되는지 살피고 담당부서가 원인을 확인하는 것부터 시작한다.

[그림 1 삽입 구역]
제목  온라인에서 끝날 일이 전화로 넘어가는 순간
구성  공식 정보 게시 → 시민 표현과 행정용어 불일치·낯선 메뉴·불명확한 안내 → 반복 검색 또는 콜센터 문의 → 담당부서 연결
표현  가운데 마찰 지점만 주황색으로 강조한다. 검색과 전화 사이에는 ‘검증할 가설, 개인 행동 연결 없음’이라고 표시한다.
캡션  김해행정마찰지도는 전화 문의로 전환될 가능성이 있는 업무를 집계 신호로 찾는다.

2. 현재 상태 - 최소 신호는 확인됐지만 연결 데이터는 미확인이다

김해시 누리집은 기간별 인기검색어와 조회수를 공개한다. 다만 조회수의 중복 집계나 봇 제외 기준은 공개자료에서 확인되지 않았다. 실증에 들어갈 때 집계 기준부터 확인해야 한다.

김해시 2024년 성과보고서에 제시된 민원상담콜센터 민원해결처리율은 61.6%다. 보고서 산식을 적용하면 공개된 반올림 수치에서 역산한 호 전환 비중은 약 38.4%다. 이 수치는 월별 총응답과 호 전환 집계가 존재한다는 사실을 보여준다. 오이관율이나 재이관율, 누리집 실패율을 뜻하지는 않는다.

검색결과 0건·클릭·재검색, 콜센터 문의유형·최초 이관·재이관 여부는 공개자료에서 확인되지 않았다. 특히 문의유형별 집계가 없으면 검색어와 전화 문의가 같은 업무를 가리키는지 대조할 수 없다. 지금 확인된 공개 신호만으로는 검색 채널의 예비진단과 시 전체 호 전환 추세 참고까지 가능하다. 두 채널을 결합한 본 실증에는 개인정보를 제거한 문의유형별 월 집계가 필요하다.

관련 계약 기반은 존재한다. 김해시 계약현황에는 2026년 누리집 통합 유지관리 용역, 2025년 민원콜센터 상담시스템 재구축 용역, 2025년 데이터 분석 사업용역이 등재돼 있다. 이는 기존 기반을 활용할 가능성을 뒷받침한다. 데이터 추출과 소규모 수정이 계약 범위에 포함되는지는 과업지시서와 담당부서 확인이 필요하다.

3. 해결방안 - 한 장의 표에서 찾고 고치고 다시 본다

김해행정마찰지도는 시민이 새로 가입하거나 설치하는 서비스가 아니다. 공무원이 매달 펼쳐 보는 한 장짜리 개선 점검표다. 표에는 업무 주제, 검색 신호, 전화 신호, 담당자가 확인한 원인, 수정 내용, 담당부서, 전후 관찰 결과를 나란히 적는다. 분석 보고서를 만드는 데서 멈추지 않고 실제 변경과 재측정까지 한 흐름으로 관리한다.

작동 순서는 여섯 단계다.

① 누리집 인기검색어·조회수와 콜센터 총응답·호 전환 집계를 같은 월 단위로 정리한다. 개인정보를 제거한 문의유형별 집계도 제공 가능한지 확인한다.

② 2주차에 데이터 게이트를 연다. 문의유형별 집계와 수정 권한을 확보하면 두 채널 결합 실증으로 진행한다. 문의유형을 확보하지 못하면 검색 채널 예비진단으로 축소하고 전화 지표는 참고자료로만 쓴다. 개인정보 보호나 계약 범위 문제를 해결하지 못하면 본 실증을 중단한다.

③ 결합 실증에서는 발생 규모와 반복성, 두 채널의 동시 신호, 2주 내 수정 가능성을 단순 규칙으로 점수화한다. 산식과 선정 이유는 점검표에 남긴다.

④ 담당자는 상위 항목이 실제 불편인지 확인한다. 행사, 제도 변경, 홍보 때문에 일시적으로 늘어난 검색은 개선 대상에서 뺀다.

⑤ 검증을 마친 3~5개 항목의 검색동의어, 메뉴명, 안내문, 자주 묻는 질문, 콜센터 연결표와 상담 매뉴얼을 고친다.

⑥ 같은 범위의 지표를 다시 살펴 유지·보완·중단을 결정한다. 전체 호 전환은 배경 추세로만 보고 관련 문의유형 지표가 있을 때만 해당 수정과 함께 해석한다.

[그림 2 삽입 구역]
제목  김해행정마찰지도의 작동 구조
구성  월별 집계 → 2주차 데이터 게이트 → 규칙 기반 후보 정렬 → AI 의미 묶기(선택) → 담당자 확인 → 누리집·연결표 수정 → 전후 관찰
표현  데이터 단계는 파란색, 사람의 판단은 짙은 회색, 실제 수정은 초록색으로 구분한다. 데이터 게이트에는 ‘진행·축소·중단’ 세 갈래를 표시한다.
캡션  데이터 조건을 먼저 확인하고 담당자가 원인을 검증한 항목만 고친다.

AI가 맡는 일

AI는 서로 다른 시민 표현을 한 주제로 묶는 보조도구다. 설명용 예시로 대형폐기물, 소파 버리는 법, 폐가구 처리는 표현이 다르지만 같은 업무 후보로 묶을 수 있다. AI는 개인정보를 제거한 검색어와 문의유형에만 적용한다. 시민 개인의 검색과 전화를 연결하거나 호 전환을 오이관으로 판정하지 않는다. 상담 메모와 녹취도 분석 대상에서 제외한다. 최종 판단은 담당 공무원이 맡는다.

4. 12주 목표 일정

1~2주에는 담당부서와 데이터 정의, 개인정보 기준, 계약 범위를 확인한다. 2주차 데이터 게이트에서 결합 실증의 진행·축소·중단을 결정한다. 진행할 경우 3~4주에 기준선을 만들고 5주에 3~5개 후보를 선정한다. 6~7주에는 메뉴·문구·검색동의어·연결표를 수정한다. 8~11주에는 같은 범위의 변화 신호를 관찰한다. 12주에는 결과를 바탕으로 유지·보완·중단을 결정한다.

12주는 확정된 소요기간이 아니라 목표 일정이다. 개인정보 검토나 데이터 추출에 추가 시간이 필요하면 항목 수를 줄이거나 예비진단으로 범위를 낮춘다.

[그림 3 삽입 구역]
제목  12주 목표 일정과 데이터 게이트
구성  준비 2주 → 진행·축소·중단 결정 → 기준선 2주 → 후보 선정 1주 → 실제 수정 2주 → 사후 관찰 4주 → 결과 정리 1주
캡션  데이터 조건을 통과한 경우에만 3~5개 항목의 수정과 전후 관찰을 진행한다.

5. 변화를 확인하는 방법

검색량이나 조회수가 줄었다고 곧바로 성공은 아니다. 홍보나 제도 변경으로 조회수가 늘어도 정확한 공식 페이지가 상위 결과에 나타나고 안내 내용이 정확하다면 접근성은 나아진 셈이다. 누리집에서는 대상 검색어의 공식 페이지 노출순위와 안내 정확성을 수동 점검한다. 이는 시민의 실제 업무 완료가 아니라 정보 도달 가능성을 보는 대리지표다. 선택 데이터가 확보되면 0건 검색과 재검색의 변화도 살핀다.

콜센터에서는 문의유형별 집계가 있을 때 관련 유형의 호 전환 건수를 관련 유형 총응답으로 나눠 비교한다. 시 전체 호 전환은 계절, 행사, 제도 변경의 영향을 받으므로 참고지표로만 둔다. 선정일부터 수정 완료일까지 걸린 기간과 실제 개선 완료율도 기록한다.

기준선이 없는 상태에서 임의의 개선률을 약속하지 않는다. 12주 실증은 작은 수정 뒤 변화 신호가 나타나는지 살피는 단계다. 인과효과를 확정하기보다 다음 실증의 기준선과 비교방법을 마련한다.

6. 실행 가능성과 비용

공개 업무분장에 따라 AI정책과 데이터융합팀을 분석 총괄 후보로 제안한다. 정보통신과는 누리집 집계와 수정, 소통공보관 시민소통팀은 콜센터 집계와 연결표 수정의 담당 후보다. 실제 주관과 협조 범위는 착수 전에 확정한다.

신규 앱이나 대시보드는 만들지 않는다. 공개 조회수, 기존 콜센터 집계, 스프레드시트, 현재 유지관리 절차를 우선 활용한다. 별도 구축비가 없는 최소형 실증부터 검토하되 추가비용을 0원으로 단정하지 않는다. 데이터 추출이나 설정 변경이 기존 계약 밖이라면 공수를 확인한 뒤 범위를 축소하거나 후속 사업으로 넘긴다. 내부 인력 투입도 필요한 자원으로 기록한다.

7. 기대효과

첫째, 시민이 새 서비스를 익히지 않아도 정확한 공식 안내에 도달하도록 돕는다. 둘째, 김해시는 시민이 직접 해결할 업무와 전문상담이 필요한 업무를 구분할 근거를 쌓는다. 셋째, 어떤 안내를 왜 고쳤고 어떤 변화 신호가 나타났는지 기록한다. 효과가 확인된 수정 절차는 다른 부서와 민원업무에도 적용하는 것을 목표로 한다.

8. 차별성

김해행정마찰지도는 AI 챗봇이나 민원 자동배정 시스템, 지리정보 지도가 아니다. 이 제안은 탐지, 담당자 확인, 실제 수정, 전후 관찰을 하나의 운영 절차로 묶는다. AI 기술 자체보다 분석 결과가 누리집과 콜센터의 실제 변경으로 이어지도록 책임과 순서를 정한 점에 차이가 있다.

마무리

김해행정마찰지도는 기술을 하나 더 얹는 제안이 아니다. 시민이 남긴 익명 신호를 읽어 운영 중인 누리집과 콜센터가 더 잘 작동하도록 고친다. 정보는 게시하는 데서 끝나지 않는다. 시민이 답에 도달할 가능성을 높이는 작은 수정부터 12주 동안 시험한다.
