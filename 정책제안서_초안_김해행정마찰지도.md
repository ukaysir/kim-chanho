## ⑦ 현황 및 문제점

### 1. 민원은 접수되기 전부터 시작된다

시민은 정식 민원을 내기 전에 누리집에서 정보를 검색하거나 콜센터에 문의한다. 이때 시민이 쓰는 생활용어와 행정용어가 다르면 원하는 안내를 찾기 어렵고, 문의 내용에 따라 담당부서로 전화가 넘어갈 수 있다. 이러한 과정에서 생기는 불필요한 재검색, 설명 반복, 잘못된 연결과 같은 작은 막힘을 이 제안에서는 **행정마찰**이라고 부른다. ‘행정마찰지도’는 시민 개인의 이동경로를 추적하는 지도가 아니라, 어느 검색어·문의 분야에서 막힘이 반복되는지를 집계한 업무 개선 목록이다.

### 2. 김해시는 분석에 활용할 수 있는 두 가지 신호를 이미 집계하고 있다

첫째, 김해시는 ‘김해 트렌드’에서 기간별 검색어와 검색 횟수를 공개한다. ‘운전면허증반납’, ‘운전면허증 환금’, ‘운전면허증 지원’, ‘운전면허증 비용지원’처럼 서로 비슷한 표현이 별개의 검색어로 나타난다. 이는 유사 표현을 우선 후보로 묶은 뒤 실제 목적이 같은지 담당자가 확인할 수 있는 기반이 있음을 보여준다. 다만 공개 화면만으로는 검색 결과 수, 결과 클릭, 재검색 로그의 보존 여부까지 확인할 수 없다.

둘째, 김해시 「2024 회계연도 성과보고서」에서 민원상담콜센터의 2024년 자체 해결률은 61.6%로 제시됐다. 보고서는 자체 해결 건수를 ‘총 응대 건수에서 담당부서 호 전환 건수를 뺀 값’으로 정의한다. 따라서 공식 산식상 전체 응대의 38.4%(100-61.6)가 담당부서로 전환된 것으로 계산된다. 다만 이 수치는 정상적인 담당부서 연결까지 포함한 전체 전환이며, 잘못된 연결이나 반복 설명의 비율을 뜻하지 않는다. 문의 분야별 전환·재전환 기록의 보유 여부도 별도 확인이 필요하다.

### 3. 현재 공개자료만으로는 ‘발견→수정→검증’의 반복 운영이 확인되지 않는다

김해시는 데이터허브와 공공 빅데이터 분석 업무를 운영해 왔으며, 2024년에는 7건의 데이터 분석 실적을 공개했다. 이러한 기존 기반은 행정마찰지도 시범의 실행 가능성을 높인다. 그러나 공개자료에서는 누리집 검색과 콜센터 전환 신호를 함께 살펴보고, 매달 하나의 시민 접점을 실제로 고친 뒤 전후 변화를 공개하는 운영 절차는 확인되지 않는다.

국민권익위원회의 ‘한눈에 보는 민원 빅데이터’는 국민신문고와 지자체에 **접수된 민원**의 현황·키워드·동향을 분석한다. 반면 시민이 정식 민원을 내기 전에 겪는 검색 막힘이나 전화 연결 과정은 김해시의 서비스 화면·안내문·연결표를 고치기 위한 별도의 관찰 대상이다. 분석자료를 만드는 데 그치지 않고 실제 서비스 수정까지 책임지는 운영 고리가 필요하다.

## ⑧ 개선방안

### 1. 제안 개요

본 제안은 새로운 시민용 앱이나 대규모 독립 플랫폼을 만드는 사업이 아니다. 김해시가 이미 보유한 범위의 검색어·콜센터 전환 데이터를 월 단위로 익명 집계하고, AI가 유사한 표현을 같은 행정 목적으로 묶도록 한다. 담당자는 AI 결과와 원자료를 검수해 개선 후보를 정하고, 매월 한 가지 화면·문구·업무매뉴얼·부서 연결표를 실제로 수정한다. 다음 달에는 수정 전후의 지표를 비교해 효과와 한계를 확인한다.

- **시범 위치:** 김해시 대표 누리집 통합검색과 민원상담콜센터의 생활민원 1개 분야
- **시범 기간(제안):** 총 12주 — 기준선·데이터 점검 4주, 개선·측정 8주
- **운영 방식:** 한 달에 한 가지 개선 과제를 선정하고, 시범기간 중 최소 2건을 수정·검증
- **사업비:** **[기존 시스템의 로그 항목, 유지관리 계약 범위, AI 공통기반 이용 가능 여부를 확인한 뒤 산정 필요]**. 시범은 별도 앱 구축보다 기존 시스템 기능개선과 내부 분석을 우선한다.
- **역할 분담(제안):** 데이터·스마트행정 담당부서가 분석을 총괄하고, 누리집·콜센터 운영부서가 데이터를 확인하며, 해당 업무 담당부서가 안내·연결 경로를 수정한다. **[실제 담당부서 지정 필요]**

### 2. 12주 시범운영 절차

#### 1단계 — 데이터 항목과 기준선 확인(1~4주)

1. 누리집에서는 현재 보유한 검색어·검색 횟수를 우선 사용한다.
2. 검색 결과 수와 결과 클릭 기록이 이미 남는다면 함께 집계한다. 남지 않는다면 개인정보를 수집하지 않는 최소 이벤트 기록의 추가 가능성을 정보보호·시스템 담당자가 먼저 검토한다.
3. 콜센터에서는 총 응대·문의 분야·담당부서 전환 항목의 보유 여부를 확인한다. 재전환 항목이 없다면 1단계 성과지표에서 제외한다.
4. 개인별 검색과 전화 기록을 연결하지 않는다. 채널별·문의 분야별 일·주 단위 집계만 사용한다.
5. 확보 가능한 지표를 4주간 측정해 기준선을 만든다. 기준선이 나오기 전에는 임의의 감소 목표를 약속하지 않는다.

#### 2단계 — 개인정보 제거와 AI 묶음 분석

1. 검색어에 포함될 수 있는 전화번호, 이메일, 주민등록번호 형식, 상세 주소 등 개인 식별 가능 표현을 규칙 기반으로 먼저 제거한다.
2. 원문은 승인된 내부 환경에서만 처리하며, 공개형 생성형 AI 서비스에 그대로 입력하지 않는다.
3. AI는 오탈자·동의어·긴 자연어 질문을 같은 행정 목적으로 묶고 각 묶음의 대표 표현을 제안한다.
4. AI는 민원 처분, 담당부서 결정, 직원 평가를 하지 않는다. 담당자가 표본을 확인해 잘못 묶인 항목을 고치고 분석 결과를 확정한다.

#### 3단계 — 매월 한 가지 개선 과제 선정

담당자는 다음 세 기준을 함께 살펴 월 1건을 선정한다.

- **발생 빈도:** 같은 목적의 검색어 수, 검색 횟수 또는 전화 전환 건수
- **시민 부담:** 결과 없음, 여러 단계 탐색, 담당부서 전환·재전환 등 확인 가능한 막힘의 정도
- **수정 가능성:** 검색 동의어·메뉴명·안내문·업무매뉴얼·부서 연결표처럼 시범기간 안에 고칠 수 있는지 여부

빈도만 높다고 자동 선정하지 않는다. 계절 행사나 일시적 이슈인지, 정상적인 전화 전환인지, 이미 수정 중인 사안인지 담당자가 함께 확인한다.

#### 4단계 — 실제 서비스 수정

선정된 원인에 따라 다음 중 하나를 시행한다.

- 시민 표현을 검색 동의어에 추가
- 찾기 어려운 메뉴명·페이지 제목 수정
- 신청대상·기한·구비서류·담당부서를 안내문 앞부분에 보강
- 콜센터 업무매뉴얼과 담당부서 연결표 현행화
- 반복 설명이 실제로 확인되고 기존 기능과 중복되지 않을 경우, 상담원이 작성한 요약을 시민이 확인한 뒤 다음 담당자에게 넘기는 ‘민원 인계패스’를 별도 개인정보 검토 후 개선 과제로 검토

마지막 항목은 전화 전환 과정의 반복 설명과 기존 상담 메모 전달 여부가 확인될 때만 적용하는 조건부 예시이며, 행정마찰지도의 필수 기능은 아니다.

#### 5단계 — 전후 측정과 공개

수정 후 4주 동안 같은 지표를 측정해 기준선과 비교한다. 개선 결과에는 좋아진 지표뿐 아니라 변화가 없거나 예상과 다른 결과도 함께 기록한다. 공개 자료에는 개인 검색어 원문, 적은 건수의 세부 집계, 직원별 수치를 포함하지 않는다. 결과는 ‘무엇을 발견했고, 무엇을 고쳤으며, 어떤 지표가 어떻게 변했는지’가 보이는 월간 개선 기록으로 남긴다.

## ⑨ 기대효과

본 제안의 효과는 아직 실증되지 않았다. 따라서 임의의 감소율을 제시하는 대신, 4주 기준선을 먼저 만들고 같은 지표를 수정 전후에 비교한다.

### 1. 시민 관점

- 시민이 사용하는 표현이 검색 동의어와 안내문에 반영돼 원하는 정보에 도달하기 쉬워질 가능성을 검증할 수 있다.
- 반복적으로 전환되는 문의 분야의 안내와 연결표를 고쳐 불필요한 전화 이동을 줄일 수 있는지 확인할 수 있다.
- 시민에게 별도의 앱 가입이나 추가 개인정보 제공을 요구하지 않고 기존 누리집과 콜센터 안에서 개선을 시도한다.

### 2. 행정 관점

- 정식 민원이 들어온 뒤 대응하는 데서 한 걸음 앞서, 검색·전화 과정의 반복 신호를 서비스 개선 과제로 전환할 수 있다.
- 담당부서는 추상적인 ‘시민 편의 향상’이 아니라 실제 검색어·전환 지표를 근거로 수정 우선순위를 정할 수 있다.
- 큰 시스템을 한 번에 바꾸지 않고, 한 분야에서 월 1건씩 고쳐 효과가 확인된 방식만 단계적으로 확산할 수 있다.

### 3. 측정 계획

| 지표 | 기준선 | 시범 목표 | 측정 방법 |
| --- | --- | --- | --- |
| 검색 결과 없음 비율 | 1~4주 측정 **[로그 보유 시]** | 기준선 확인 후 설정 | 전체 검색 중 결과 0건 비율을 수정 전후 비교 |
| 개선 대상 검색어의 목적 페이지 도달 | 1~4주 측정 **[클릭 로그 보유 시]** | 기준선 확인 후 설정 | 대상 검색어 입력 후 목적 페이지 클릭 비율 비교 |
| 선택 분야의 담당부서 전환율 | 1~4주 측정 **[분야 항목 보유 시]** | 기준선 확인 후 설정 | 해당 분야 총 응대 중 부서 전환 건수 비교 |
| 재전환율 | 1~4주 측정 **[재전환 항목 보유 시]** | 기준선 확인 후 설정 | 최초 전환 뒤 다시 다른 부서로 전환된 비율 비교 |
| 개선 완료 건수 | 시범 시작 시 0건 | 12주간 최소 2건 | 수정 전·후 화면, 안내문 또는 연결표와 완료일 기록 |
| AI 분류 수정률 | 첫 분석 때 측정 | 지속 관찰 | AI가 묶은 표본 중 담당자가 수정한 비율 기록 |

정량지표가 좋아져도 계절·홍보·제도 변경의 영향을 함께 기록해 제안만의 효과로 과장하지 않는다. 반대로 변화가 없으면 수정 내용을 되돌리거나 다른 원인 가설을 시험한다. 이렇게 쌓인 월간 개선 기록은 김해시가 시민의 실제 이용 언어와 행동을 반영해 서비스를 계속 고치는 근거가 된다.

## 1. 작동 예시: 같은 뜻의 여러 검색어를 하나의 개선 과제로 묶기

김해 트렌드에는 ‘운전면허증반납’, ‘운전면허증 환금’, ‘운전면허증 지원’, ‘운전면허증 비용지원’, ‘운전면허시험지원금’처럼 비슷해 보이지만 실제 목적은 다를 수 있는 검색어가 나타난다. 행정마찰지도는 다음 순서로 작동한다.

`시민 검색 표현 수집 → 개인정보 제거 → AI가 유사 목적 후보로 묶음 → 담당자가 ‘면허 반납 지원’과 ‘면허시험 지원’처럼 실제 목적이 다른 항목을 분리 → 검색 결과와 안내 페이지 확인 → 필요한 동의어·제목·안내문 수정 → 4주 뒤 같은 지표 비교`

이 예시는 해당 검색이 모두 실패했다는 뜻이 아니다. 서로 다른 시민 표현을 하나의 점검 과제로 묶는 방식을 설명한다.

# 원고 밖 — 제출 전 확인할 항목

아래 내용은 확인 후 초안에 반영하고, 제출본에서는 삭제한다.

## P0 — 제출·중복 위험

- [ ] 제안자가 동일·유사 아이디어를 다른 행정청이나 공모전에 제출했는지 확인하고 ⑥ 항목에 표시
- [ ] 김해시가 검색·전화의 사전 행정마찰을 묶어 월 1건 수정하는 동일·유사 사업을 이미 시행·구축 중인지 정책기획과에 확인
- [ ] 2025년 김해시 데이터 분석 사업과 2026년 현재 추진사업에 검색 로그·콜센터 전환 분석 과제가 포함됐는지 확인
- [ ] 공고 이후 정정·연장 공지가 게시됐는지 제출 당일 다시 확인

## P1 — 실행가능성 보강

- [ ] 누리집 로그에 검색 결과 수와 결과 클릭 항목이 남는지 확인
- [ ] 콜센터 데이터에 문의 분야, 최초 전환, 재전환 항목이 있는지 확인
- [ ] 시범 운영을 맡을 총괄·협력부서와 기존 시스템 유지관리 계약 범위 확인
- [ ] 기존 인프라 활용 범위와 필요한 기능개선 항목을 확인한 뒤 사업비 산정
- [ ] 개인정보 제거 방식과 AI 처리 환경을 개인정보보호·정보보안 담당자가 검토해야 한다는 절차 반영

## P2 — 설득력 보강

- [ ] 시민 3~5명에게 제목과 ⑤ 주요내용만 보여주고 ‘무엇을 어떻게 고치는 제안인지’ 자기 말로 설명하게 하는 이해도 확인
- [ ] 검색 또는 전화 중 시범대상 생활민원 1개 분야를 기준선 자료로 선정
- [ ] 실제 담당자에게 월 1건 수정 주기의 업무 부담과 가능한 개선 항목 확인
